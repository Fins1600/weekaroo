# Contributing

Thanks for helping improve Weekaroo.

## Development principles

- Keep the default experience generic and family-safe.
- Do not hardcode personal names, teams, addresses, calendar URLs, API keys, or tokens.
- Put user-specific behavior in config examples or documented settings, not source defaults.
- Prefer local-first behavior and clear privacy warnings.

## Pull request checklist

- [ ] No private data or secrets in source, docs, screenshots, or examples
- [ ] Example config still works as a starter
- [ ] `python3 check_server.py` passes
- [ ] README updated for user-facing changes
